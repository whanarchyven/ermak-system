# Feature Sliced Design Architecture

Проект реорганизован согласно архитектуре Feature Sliced Design (FSD).

## Структура папок

```
├── app/                    # Next.js App Router
├── convex/                 # Convex backend
├── components/             # shadcn/ui components
├── entities/               # Бизнес-сущности
│   └── category/          # Сущность категории
│       ├── model/         # Типы и модели
│       ├── ui/            # UI компоненты
│       └── index.ts       # Экспорты
├── features/              # Функциональные возможности
│   ├── navigation/        # Навигация
│   └── category-management/ # Управление категориями
├── widgets/               # Композитные блоки
│   ├── header/           # Заголовок с навигацией
│   └── layout/           # Общий layout
├── shared/               # Общие ресурсы
│   ├── ui/               # Переиспользуемые UI компоненты
│   ├── lib/              # Утилиты
│   ├── api/              # API утилиты
│   └── config/           # Конфигурация
└── pages/                # Страницы (зарезервировано)
```

## Слои архитектуры

### 1. **app/** - Next.js App Router
- Страницы приложения
- Роутинг
- Layout компоненты

### 2. **entities/** - Бизнес-сущности
- **category/** - Сущность категории
  - `model/types.ts` - Типы данных
  - `ui/CategoryCard.tsx` - Компонент карточки категории
  - `index.ts` - Экспорты

### 3. **features/** - Функциональные возможности
- **navigation/** - Навигация
  - `ui/Navbar.tsx` - Компонент навигации
  - `index.ts` - Экспорты
- **category-management/** - Управление категориями
  - `model/useCategories.ts` - Хук для работы с категориями
  - `index.ts` - Экспорты

### 4. **widgets/** - Композитные блоки
- **header/** - Заголовок
  - `ui/Header.tsx` - Компонент заголовка с навигацией
  - `index.ts` - Экспорты
- **layout/** - Общий layout
  - `ui/Layout.tsx` - Общий layout для страниц
  - `index.ts` - Экспорты

### 5. **shared/** - Общие ресурсы
- **ui/** - Переиспользуемые UI компоненты
  - `StatisticsCard.tsx` - Карточка статистики
  - `index.ts` - Экспорты
- **lib/** - Утилиты
- **api/** - API утилиты
- **config/** - Конфигурация

## Принципы FSD

### 1. **Слоистая архитектура**
- Каждый слой может импортировать только слои ниже себя
- `app` → `pages` → `widgets` → `features` → `entities` → `shared`

### 2. **Изоляция фич**
- Каждая фича изолирована и может быть легко удалена
- Фичи не зависят друг от друга

### 3. **Переиспользование**
- `shared` содержит общие компоненты и утилиты
- `entities` содержат бизнес-логику
- `features` содержат пользовательские сценарии

### 4. **Модульность**
- Каждый модуль имеет свой `index.ts` для экспортов
- Четкие границы между модулями

## Примеры использования

### Импорт компонентов
```typescript
// Импорт сущности
import { CategoryCard } from "../../entities/category";

// Импорт фичи
import { useCategories } from "../../features/category-management";

// Импорт виджета
import { Layout } from "../../widgets/layout";

// Импорт shared компонентов
import { StatisticsCard } from "../../shared/ui";
```

### Создание новой фичи
1. Создать папку в `features/`
2. Добавить `ui/`, `model/`, `api/` папки при необходимости
3. Создать `index.ts` для экспортов
4. Использовать в страницах

## Преимущества новой архитектуры

✅ **Масштабируемость** - Легко добавлять новые фичи
✅ **Поддерживаемость** - Четкая структура и изоляция
✅ **Переиспользование** - Общие компоненты в shared
✅ **Тестируемость** - Изолированные модули легче тестировать
✅ **Командная работа** - Разные разработчики могут работать над разными фичами
